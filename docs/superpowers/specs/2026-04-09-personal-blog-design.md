# Cloudflare 个人博客设计文档

**日期：** 2026-04-09  
**主题：** 可部署到 Cloudflare 的个人博客网站

## 一、项目目标

构建一个适合部署在 Cloudflare Pages 上的个人博客网站，第一版同时满足以下两个目标：

- 作为技术博客发布和展示文章
- 作为个人主页展示个人信息、项目作品、技术栈和联系方式

该网站第一版应优先保证：

- 页面简洁、干净、偏技术博客风格
- 内容维护简单
- 构建与部署稳定
- 页面访问速度快
- 后续具备扩展空间

## 二、方案结论

第一版采用以下技术方案：

- **框架：** Astro
- **文章管理：** Markdown
- **项目与站点数据：** 本地数据文件
- **部署平台：** Cloudflare Pages

不采用在线后台，也不在第一版引入数据库、用户系统和原生评论系统。

**Why:** 当前目标是尽快交付一个稳定、好维护、适合长期写作和个人展示的网站，而不是一开始就做成复杂的全栈应用。  
**How to apply:** 所有结构设计、页面设计和实现边界都应围绕“轻量、静态优先、内容优先”展开。

## 三、信息架构

网站分为两大部分：

### 1. 博客内容区
- 首页文章入口
- 文章详情页
- 文章归档页
- 标签页
- RSS

### 2. 个人展示区
- 首页个人简介
- 首页技术栈
- 首页精选项目
- 项目详情页
- 关于我页面
- 联系方式页面

首页作为统一入口，既承担个人展示，也承担内容分发职责。

## 四、页面设计

### 1. 首页
首页按以下顺序组织内容：

1. 顶部导航
2. 首屏个人简介
3. 技术栈
4. 精选项目
5. 置顶推荐文章
6. 最新文章列表
7. 联系方式
8. 页脚

这样能让访客先认识你，再了解你做过什么，最后自然进入文章内容。

### 2. 文章详情页
第一版包含以下内容：

- 标题
- 发布时间
- 阅读时长
- 标签
- 文章目录
- Markdown 正文渲染
- 代码高亮
- 上一篇 / 下一篇导航
- 返回文章列表
- 预留评论区位置

### 3. 文章归档页
按时间倒序展示全部文章，重点保证查找效率和结构清晰。

### 4. 标签页
展示全部标签，以及各标签下的文章集合。第一版仅做标签体系，不额外增加复杂分类系统。

### 5. 项目页
首页展示精选项目，点击后进入项目详情页。项目详情页包括：

- 项目名称
- 项目简介
- 技术栈
- 项目截图
- 项目亮点
- 项目演示地址
- 仓库地址

### 6. 关于我页面
展示完整个人介绍、技术方向、经验总结和博客定位。

### 7. 联系方式页面
第一版仅展示：

- 邮箱
- GitHub

## 五、内容模型设计

### 1. 博客文章
使用 Markdown 文件管理，每篇文章包含：

- title
- description
- pubDate
- tags
- featured
- draft
- cover（可选）

### 2. 项目数据
使用本地数据文件管理，每个项目包含：

- slug
- title
- description
- techStack
- cover
- highlights
- demoUrl
- repoUrl
- featured

### 3. 站点信息
站点标题、个人简介、社交方式、技术栈等统一放在配置文件中集中管理。

## 六、视觉与交互设计

### 1. 风格原则
- 简洁干净
- 重视阅读体验
- 留白适中
- 强调层级和排版
- 整体偏技术博客风格

### 2. 交互范围
第一版支持：

- 深色模式切换
- 移动端自适应
- 标签与链接的悬停反馈
- 清晰的代码块样式
- 统一的按钮与卡片样式

不引入复杂动画和大面积交互特效。

## 七、Cloudflare 部署设计

部署方式为：

- 源码托管在代码仓库
- Astro 构建生成静态页面
- Cloudflare Pages 自动构建和部署

第一版不依赖数据库和服务端渲染能力，优先以静态生成方式完成全部核心页面。

## 八、边界与错误处理策略

第一版应覆盖以下边界情况：

- 无文章时首页和文章列表展示空状态
- 无项目时项目区展示空状态
- 标签不存在时显示友好提示
- 未找到页面时显示 404 页面
- 图片缺失时不破坏整体布局
- 首次访问时深色模式优先跟随系统设置

## 九、测试与验收标准

### 基础验收
- 本地可以正常启动
- 项目可以正常构建
- 可部署到 Cloudflare Pages
- 页面在桌面端和移动端可正常访问

### 功能验收
- 首页同时展示个人信息、项目和文章入口
- Markdown 文章可以正确渲染
- 标签页、归档页、项目详情页、关于页、联系页可访问
- 深色模式切换正常
- RSS 可正常生成

### 明确不做
- 在线后台
- 原生评论系统
- 用户登录系统
- 数据库接入
- 复杂服务端逻辑

## 十、最终范围

第一版最终交付范围为：

- 首页
- 文章详情页
- 文章归档页
- 标签页
- 项目详情页
- 关于我页面
- 联系方式页面
- 深色模式
- RSS
- 代码高亮
- 文章目录
- 上一篇 / 下一篇导航
- 404 页面

该范围适合作为个人博客的第一版上线版本，既能满足长期写作，也能支撑个人展示，并且非常适合部署到 Cloudflare Pages。
