## 为什么 Markdown 仍然是个人博客的好选择

对于以文字输出为核心的站点，Markdown 依然是最稳定、最轻量的方案。它没有复杂后台，也不会把写作和展示逻辑强行绑在一起。

### 文件即内容

文章就是普通文本文件，可以直接进入版本控制，也方便迁移和备份。这一点对长期维护非常重要。

### 工具链足够成熟

无论是本地编辑器、GitHub 仓库还是静态站点生成器，对 Markdown 的支持都已经非常完整。

## 我习惯的发布流程

先写标题、摘要和标签，再补正文结构，最后统一检查链接和代码块。这样做的好处是页面摘要、归档页和搜索结果都会更早成形。

### 发布前检查目录结构

如果文章里有多个二级、三级标题，就顺手确认一下文章目录是否清晰。目录不是装饰，它决定了长文能不能顺畅阅读。

### 发布前检查摘要与标签

标签太多会让归档页显得杂乱，摘要太泛会让列表页失去辨识度。每篇文章尽量只保留真正有区分度的信息。

## 写作系统要服务于更新频率

博客的目标不是把每篇文章都写成文档，而是建立一个可以持续更新的表达系统。只要写作流程足够稳定，文章数量自然会逐步增长。

## 结语

Markdown 最有价值的地方，不是语法简单，而是它能让写作这件事保持专注、轻量并且长期可持续。
